Overview

Common base view controller that all full-screen view controllers should inherit from. Takes care of certain boilerpoint like automatically logging when VC appears.

Implements the protocol, which means you can use XNavigationService to push an XViewController with [XNavigationService navigateTo:params:]. The params property is automatically set and logged.

XViewController also automatically adds the call [NSNotificationCenter.defaultCenter removeObserver:self] in dealloc Check and make sure various calls in the view lifcycle are properly forwarded to super by doing some checking here

Class Methods

validateParams:

Optional class method that can be used to do parameter validation. Default implementation does nothing, but subclasses of XViewController can override this to validate their parameters. This method is called, if it exists, whenever the view controller is pushed. Use NSAssert/NSParameterAssert to do simple parameter checking.

+ (void)validateParams:(NSDictionary *)params

initWithParams:

Initializes a new XViewController with a dictionary of parameters. You should not call this directly - XNavigationService will call this automatically. Subclasses of XViewController should override [super initWithParams:] rather than init: or initWithNibNamed:.

- (instancetype)initWithParams:(NSDictionary *)params

setup

This method is called whether the view controller is created with init, initWithNibName:bundle: initWithParams:, or via a nib file (awakeFromNib). In any of the above cases, [self setup is called]. You can override setup to do logic that would otherwise go in an init method (and possibly be duplicated amongst several). The default implementation does nothing.

- (void)setup
